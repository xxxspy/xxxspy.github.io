    Cohen’s Kappa 系数：是什么以及为什么？

    作者：Ruben Geert van den Berg，归类于 Statistics A-Z

    Cohen’s Kappa 系数是一种用于衡量两个评分者之间的评分一致性是否优于偶然情况的指标。

    Cohen’s Kappa - 快速示例

    假设两名儿科医生观察了 N = 50 名儿童。他们独立地对每个孩子进行诊断。获得的数据在 这个 Google 表格 中，部分内容如下所示。

    我们很容易看到，我们的评分者在某些儿童的诊断上一致，而在另一些儿童的诊断上不一致。所以我们想知道的是：我们的评分者在多大程度上对这些诊断结果达成一致？一个显而易见的方法是计算评分者达成一致的儿童比例。我们可以很容易地通过创建一个列联表或“交叉表”（crosstab）来实现，如下所示。

    请注意，绿色对角线元素是评分者达成一致的儿童数量。观察到的一致性比例 \(P_a\) 很容易计算为：

    \[P_a = \frac{27 + 2 + 2 + 2 + 1}{50} = 0.68\]

    这意味着我们的评分者对 50 个孩子中的 68% 给出了相似的诊断。现在，这看起来可能相当不错，但是如果我们的评分者通过简单地抛硬币来诊断儿童是否（不）健康会怎么样？这样的诊断毫无价值，对吧？尽管如此，在这种情况下，我们预计一致性比例 \(P_a\) = 0.50：我们的评分者仅仅通过偶然的机会就能在 50% 的儿童诊断上达成一致。

    解决这个问题的方法是校正这种偶然水平的一致性比例。Cohen’s Kappa 正是这样做的。

    Cohen’s Kappa - 公式

    首先，有多少儿童被诊断为相似？为此，我们只需将下表中对角线上的元素（绿色）相加。

    结果是：

    \[\Sigma{o_{ij}} = 27 + 2 + 2 + 2 + 1 = 34\]

    其中 \(o_{ij}\) 表示对角线上的观察频率（observed frequencies）。

    其次，如果我们的评分者根本没有达成任何一致，我们应该期望有多少相似的诊断结果仅仅是出于偶然？这些预期频率计算如下：

    \[e_{ij} = \frac{o_i\cdot o_j}{N}\]

    其中：

    • \(e_{ij}\) 表示一个预期频率（expected frequency）；
    • \(o_i\) 是相应的行边际频率（marginal row frequency）；
    • \(o_j\) 是相应的列边际频率（marginal column frequency）；
    • \(N\) 是总样本大小。

    例如，对于评分者 A = “精神健康”（n = 35）和评分者 B = “精神健康”（n = 33）的预期频率是：

    \[e_{ij} = \frac{35\cdot 33}{50} = 23.1\]

    下表显示了这个以及所有其他预期频率。

    绿色对角线元素显示了两个评分者仅凭偶然机会给出相似诊断的所有预期频率。这些加起来是：

    \[\Sigma{e_{ij}} = 23.1 + 0.72 + 0.32 + 0.30 + 0.08 = 24.52\]

    最后，Cohen’s Kappa（表示为 \(\)，希腊字母 kappa）的计算公式为：

    \[\kappa = \frac{\Sigma{o_{ij}} - \Sigma{e_{ij}}}{N - \Sigma{e_{ij}}}\]

    对于我们的示例，结果是：

    \[\kappa = \frac{34 - 24.52}{50 - 24.52} = 0.372\]

    正如 我们的 Google 表格 中确认的那样，如下所示。

    Cohen’s Kappa 的另一种公式是：

    \[\kappa = \frac{P_a - P_c}{1 - P_c}\]

    其中：

    • \(P_a\) 是在我们的数据中观察到的一致性比例；
    • \(P_c\) 是仅仅出于偶然可能预期的一致性比例。

    对于我们的数据，结果是：

    \[\kappa = \frac{0.68 - 0.49}{1 - 0.49} = 0.372\]

    这个公式也阐明了 Cohen’s Kappa 的真正含义：

    \[\kappa = \frac{\text{实际表现 - 偶然表现}}{\text{完美表现 - 偶然表现}}\]

    归结为：

    \[\kappa = \frac{\text{相对于偶然的实际改进}}{\text{相对于偶然的最大可能改进}}\]

    Cohen’s Kappa - 解释

    正如我们刚才看到的，Cohen’s Kappa 基本上表明了观察到的一致性在多大程度上优于偶然一致性。从技术上讲，一致性也可能 低于 偶然性，导致 Cohen’s Kappa < 0。简而言之，Cohen’s Kappa 的取值范围可以从 -1.0 到 1.0（包括两者），其中：

    • \(\) = -1.0 表示 2 个评分者完全 不 一致；
    • \(\) = 0.0 表示 2 个评分者的一致性处于偶然水平；
    • \(\) = 1.0 表示 2 个评分者完全一致。

    另一种考虑 Cohen’s Kappa 的方式是与偶然性相比，不一致性减少的比例。对于我们的示例，我们预计偶然情况下有 N = 25.48 个 不同的 诊断。由于 \(\) = .372，因此减少到：

    \[25.48 - (25.48 \cdot 0.372) = 16\]

    我们的评分者给出的不同（不一致）的诊断。

    关于 效应量，对于经验法则没有明确的共识。但是，Twisk (2016) 或多或少地提出：

    • \(\) = 0.4 表示小效应；
    • \(\) = 0.55 表示中等效应；
    • \(\) = 0.7 表示大效应。

    何时（不）使用 Cohen’s Kappa？

    Cohen’s Kappa 主要适用于比较两个评分，如果两个评分都：

    • 是名义变量（nominal variables）（未 排序的答案类别）并且
    • 具有相同的答案类别。

    对于有序或定量变量，Cohen’s Kappa 不是最佳选择。这是因为它仅区分评分的一致或不一致。因此，假设我们有如下答案类别：

    1. 非常差;
    2. 有点差;
    3. 中性;
    4. 有点好;
    5. 非常好。

    如果 2 个评分者将某个项目评为“非常差”和“有点差”，他们会 稍微 不一致。如果他们将其评为“非常差”和“非常好”，他们的分歧会更强烈，但是 Cohen’s Kappa 忽略了这种重要的差异：在两种情况下，他们都只是“不一致”。一个 确实 考虑了评分者之间的不一致程度的指标是加权 Kappa（weighted kappa）。因此，这是一种更适合有序变量的指标。

    相关指标

    Cohen’s Kappa 是 2 个名义变量的关联性指标。为了测试这种关联性是否为零（两个变量是否独立），我们经常使用 卡方独立性检验。此测试的效应量指标是：

    • 列联系数（contingency coefficient）；
    • Cramér’s V；
    • Cohen’s W。

    这些指标基本上可以看作是名义变量的 相关性。那么它们与 Cohen’s Kappa 有什么不同呢？好吧，

    • Cohen’s Kappa 修正了偶然水平的一致性，并且
    • Cohen’s Kappa 要求两个变量都具有相同的答案类别，

    而其他指标则不需要。

    其次，如果两个评分都是有序的（ordinal），那么加权 Kappa 比 Cohen’s Kappa 更合适。该指标考虑了（或“加权”）评分者之间 不一致的程度。加权 Kappa 在 SPSS 27 版本中引入，位于 分析 (Analyze) 尺度 (Scale) 加权 Kappa (Weighted Kappa) 下，如下所示。

    最后，如果您有 3 个或更多评分者（3(+) raters）而不是只有两个，请使用 Fleiss 多评分者 Kappa（Fleiss-multirater-kappa）。此指标在 SPSS 28(+) 版本中可用，位于 分析 (Analyze) 尺度 (Scale) 可靠性分析 (Reliability Analysis) 下。请注意，这与用于 Cronbach’s alpha 的对话框相同。
