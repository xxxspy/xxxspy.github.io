    作者：Ruben Geert van den Berg，发布于 回归分析

    示例

    一位科学家想了解哪些因素会影响患病人群的总体幸福感。为了找到答案，她收集了一个由 N = 421 名癌症患者组成的样本数据。这些数据（部分如下所示）位于 wellbeing.sav 文件中。

    现在，我们的科学家认为幸福感受到疼痛和疲劳的影响。最重要的是，她认为疲劳本身也受到疼痛的影响。简而言之：疼痛通过疲劳部分地影响幸福感。也就是说，疲劳中介了疼痛对幸福感的影响，如下图所示。

    下半部分展示了一个模型，其中疲劳被（错误地）忽略了。这被称为“总效应模型”，通常与上面的中介模型进行比较。

    如何检验中介效应？

    现在，让我们假设一下，我们科学家的所有期望都是完全正确的。如果是这样，那么我们应该在数据中看到什么？经典的中介方法（参见 Kenny & Baron, 1986）认为：

    • \(a\) (从疼痛到疲劳) 应该是显著的；
    • \(b\) (从疲劳到幸福感) 应该是显著的；
    • \(c\) (从疼痛到幸福感) 应该是显著的；
    • \(c,’\) (直接效应) 应该比 \(c\) (总效应) 更接近于零。

    那么如何确定我们的数据是否符合这些陈述呢？好吧，所有路径在技术上都只是 b 系数。因此，我们将运行 3 个（独立的）回归分析：

    • 疼痛对疲劳的回归告诉我们 \(a\) 是否显著；
    • 多元线性回归 从疼痛和疲劳到幸福感告诉我们 \(b\) 和 \(c,’\) 是否显著；
    • 疼痛对幸福感的回归告诉我们 \(c\) 是否显著和/或与 \(c,’\) 不同。

    SPSS 中介效应分析输出

    对于我们的中介效应分析，我们真正只需要 3 个系数表。我将它们复制粘贴到 这个 Googlesheet 中（只读，部分如下所示）。

    那么我们得出什么结论呢？好吧，我们的结果满足了中介的所有要求：

    • 效应 \(a\)、\(b\) 和 \(c\) 都是 统计显著的。这是因为它们的“Sig.”或 p < .05；
    • 直接效应 \(c,’\) = -0.17，因此比总效应 \(c\) = -0.22 更接近于零。

    下图总结了这些结果。

    请注意，\(c\) 和 \(c,’\) 都是显著的。这通常被称为部分中介：疲劳部分地中介了疼痛对幸福感的影响：添加疲劳会降低效应，但不会完全消除它。

    除了部分中介外，我们有时会发现完全中介。这意味着 \(c\) 是显著的，但 \(c,’\) 不是显著的：该效应被完全中介，因此当将中介变量添加到回归模型中时，该效应消失。

    APA 格式报告中介效应分析

    中介效应分析通常作为单独的回归分析报告，例如“我们分析的第一步表明，疼痛对疲劳的影响是显著的，b = 0.09, p < .001…” 一些作者还包括 b 系数的 t 值和自由度 (df)。由于某些非常愚蠢的原因，SPSS 不报告自由度，但您可以将其计算为

    \[df = N - k - 1\]

    其中

    • \(N\) 表示总样本大小（在我们的示例中 N = 421），以及
    • \(k\) 表示模型中预测变量的数量（在我们的示例中为 1 或 2）。

    像这样，我们可以报告“我们分析的第二步表明，疲劳对幸福感的影响也是显著的，b = -0.53, t (419) = -3.89, p < .001…”

    后续步骤 - Sobel 检验

    在我们的分析中，疼痛通过疲劳对幸福感的间接效应由两个单独的效应组成，\(a\)（疼痛对疲劳）和 \(b\)（疲劳对幸福感）。现在，整个间接效应 \(ab\) 只是简单地计算为

    \[\text{indirect effect} \;ab = a \cdot b\]

    这是完全有道理的：如果工资 \(a\) 是每小时 30 美元，而税收 \(b\) 是每美元收入 0.20 美元，那么我每小时将支付 30 美元 · 0.20 美元 = 6.00 美元的税，对吗？

    对于我们的示例，\(ab\) = 0.09 · -0.53 = -0.049：对于疼痛每增加一个单位，幸福感平均通过疲劳减少 0.049 个单位。但是我们如何获得此间接效应的 p 值和 置信区间？ 有 2 个基本选项：

    • 现代文献倾向于使用 PROCESS 宏 中实现的 bootstrapping，我们将在后面讨论；
    • Sobel 检验（也称为“正态理论”方法）。

    第二种方法假设 \(ab\) 服从 正态分布，其中

    \[se_{ab} = \sqrt{a^2se^2_b + b^2se^2_a + se^2_a se^2_b}\]

    其中

    \(se_{ab}\) 表示 \(ab\) 的标准误差，依此类推。

    那么这告诉我们什么呢？好吧，我们的间接效应是显著的，B = -0.049, p = .002, 95% CI [-0.08, -0.02]。

    后续步骤 - 中介效应指数

    我们的研究变量（如疼痛和疲劳）是在没有明确测量单位的不同尺度上测量的。这使得比较它们的影响变得不可能。解决方案是报告称为 β（希腊字母“beta”）的标准化系数。

    我们的 SPSS 输出已经包括大多数效应的 beta，但不包括 \(ab\)。但是，我们可以轻松地将其计算为

    \[\beta_{ab} = \frac{ab \cdot SD_x}{SD_y}\]

    其中

    \(SD_x\) 是我们的 X 变量的样本标准差，依此类推。

    这种标准化的间接效应称为中介效应指数。为了计算它，我们可以在 SPSS 中运行类似 DESCRIPTIVES pain wellb. 的命令。将生成的表格复制粘贴到 这个 Googlesheet 后，我们将使用一个快速公式计算 \(_{ab}\)，如下所示。

    最终说明

    在 SPSS 中进行中介效应分析可以使用或不使用 PROCESS 宏。不 使用 PROCESS 的一些原因是：

    • 许多人发现 PROCESS 难以使用，并且不喜欢其输出格式；
    • PROCESS 无法创建回归残差和相关的图，用于检查回归假设，例如线性、同方差性和正态性；
    • PROCESS 输出不包括调整后的 r 平方；
    • PROCESS 不提供 缺失值 的成对排除。

    那么为什么有人使用 PROCESS 呢？一些原因可能是：

    • PROCESS 使用 bootstrapping 而不是 Sobel 检验。据说这会导致更高的 功效 和更准确的置信区间。可悲的是，bootstrapping 不会产生间接效应的 p 值，而 Sobel 检验会；
    • 对于更复杂的模型（平行、串行和调节中介），使用 PROCESS 可以节省大量工作；
    • 如果需要，PROCESS 可以处理 X 变量和调节变量（如果有）的 虚拟编码；
    • PROCESS 不需要我们在 Googlesheet 中实现的额外计算：它一次性计算你需要的所有内容。
